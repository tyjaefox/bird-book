# Bird Book (catchy I know)

A practical tool I made for viewing aircraft records for a fleet of 
UH-60's. It outputs fault history and
upcoming maintenance (hours / days). This started as a personal tool to avoid
using the enterprise application for DOD aircraft records, but should be shared as it
is much simpler and faster when planning activities weeks or even months ahead of time. 

Built with Python, Flask, SQLite, and SQL. Could have improved or used a JS framework, 
but I was more familiar with Python at the time of making it. 

> **None of the data in this repo is real.** All aircraft
> data used is generated by running `generate_data.py`. My biggest concern when
> making this publicly available was being able to demonstrate the program
> while not being able to use real data. Although the tail numbers seem realistic or 
> might exist somewhere in the real world, the data in this repo is not related in 
> any way to an existing aircraft.  Component names and
> fault wording are realistic so the ouput reads
> realistically, but are completely fake.


## Features

- Aircraft search — find a tail by tail number, model, or unit (this can be iffy).
- Per-aircraft history — full fault log with corrective actions, downtime, and open/closed status.
- Upcoming maintenance — lists scheduled maintenance, with remaining hours and remaining days, color-coded DUE / SOON / OK to easily see the status when planning. 
- Fleet analytics — top faults, components with the most failures, highest-downtime aircraft, and recurring-issues.
- Keyword fault search — scan the whole log for common terms.
- Readiness summary — FMC / PMC / NMC counts across the fleet.


Only dependency is Flask — everything else is the Python standard library.

Open `http://localhost:5000`. The app binds to `0.0.0.0`, so other people on
the same network can reach it at `http://<your-ip>:5000`.


## Data model

Five tables keep history and forward-looking maintenance separate but linked:

- **aircraft** — tail number, model, unit, airframe hours, utilization rate, readiness status.
- **components** — catalog of trackable parts/systems (drivetrain, rotor, powerplant, hydraulics, electrical, flight controls, APU).
- **aircraft_components** — a specific component on a specific aircraft with hours-since-new and life limit → drives **remaining hours**.
- **inspections** — scheduled inspections per aircraft, hours- and/or calendar-driven → drives **remaining hours/days**.
- **discrepancies** — the fault log: date, component, discrepancy, corrective action, downtime, status.


## Tweaking the data

You can adjust the number of aircraft/faults generated by tweaking the values in generate_data.py. 
By defauly, it generates 80 aircraft, but for realism, you could lower this and the number of organization units 
and increase the number of faults. This would better reflect a real scenario of having few aircraft over a longer period of time. 

Having more aircraft just works better for testing and showing off the script.
